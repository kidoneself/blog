## 📝 如何创建新文章

### 1. 选择分类

确定文章属于哪个分类：
- 技术相关 → `posts/tech/`
- 生活记录 → `posts/life/`
- 工作经验 → `posts/work/`

### 2. 创建文件

```bash
# 技术文章
touch posts/tech/my-new-article.md

# 生活随笔
touch posts/life/my-life-story.md

# 工作感悟
touch posts/work/work-experience.md
```

### 3. 使用模板

复制对应分类的写作模板，填写内容。

### 4. 更新配置

编辑 `.vitepress/config.mts`，在对应的 sidebar 中添加新文章：

```typescript
sidebar: {
  '/posts/tech/': [
    {
      text: '技术文章',
      items: [
        // 添加新文章
        { text: '新文章标题', link: '/posts/tech/my-new-article' },
      ]
    }
  ]
}
```

### 5. 更新文章列表

编辑 `posts/index.md` 和对应分类的 `index.md`，添加新文章链接。

## 🎯 内容规划建议

### 保持平衡

```
技术 : 生活 : 工作 = 4 : 3 : 3
```

- **技术文章**：保持技术深度和专业性
- **生活随笔**：记录真实感受，不追求数量
- **工作感悟**：提炼可复用的经验

### 定期更新

- **每周至少 1 篇文章**
- **每月至少涵盖所有分类**
- **季度回顾和总结**

### 质量优先

- ✅ 内容充实，有深度
- ✅ 结构清晰，易阅读
- ✅ 配图和代码示例
- ✅ 可操作的建议
- ❌ 避免流水账
- ❌ 避免过于简短

## 💡 写作技巧

### 1. 开头吸引人

```markdown
❌ "今天我要讲..."
✅ "你是否遇到过这样的问题..."

❌ "这篇文章介绍..."
✅ "3 个技巧让你的代码性能提升 50%"
```

### 2. 结构清晰

- 使用清晰的标题层级
- 适当的列表和表格
- 代码块和示例
- 提示框（tip、warning、danger）

### 3. 配图和示例

- 流程图和架构图
- 代码示例
- 截图和动图
- 对比表格

### 4. 总结和行动

每篇文章最后：
- 核心要点总结
- 可执行的行动项
- 延伸阅读资源
